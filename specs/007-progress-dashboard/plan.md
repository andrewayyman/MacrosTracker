**Branch**: `007-progress-dashboard` | **Date**: 2026-05-27 | **Spec**: [spec.md](spec.md)

**Input**: Feature specification from `/specs/007-progress-dashboard/spec.md`

## Summary

Build a read-only Progress Dashboard that closes the feedback loop between food logging (Specs 004вЂ“006) and goal setting (Spec 003). The frontend gains a new `/progress` page with four views: daily macro progress bars, 7/30/90-day trend charts, streak + heatmap, and weekly summary. The backend gains a new `ProgressController` backed by a `ProgressService` that computes all analytics at read time from existing `MealLog` and `DailyNutritionGoal` entities вЂ” no new database schema is required.

## Technical Context

**Language/Version**: C# / .NET 10 (backend) В· React 18 with Vite + JSX (frontend)

**Primary Dependencies**: ASP.NET Core В· EF Core В· SQL Server (backend); React Router В· Axios В· TanStack Query В· Recharts (frontend вЂ” install if not present)

**Storage**: SQL Server. No new tables or migrations. Queries aggregate existing `MealLog` (by `UserId` + `DiaryDate`) against `DailyNutritionGoal` (active row per user).

**Testing**: None вЂ” not requested in spec (Constitution Principle V).

**Target Platform**: Web (SPA + REST API)

**Performance Goals**: Dashboard page load < 2 s; trend chart load < 3 s under normal network conditions.

**Constraints**: Authenticated users only. Read-only. No new stored entities.

**Scale/Scope**: Per-user analytics over up to 90 days of log data.

## Implementation Phases

### Phase 1 вЂ” Backend: ProgressService and Controller

**Goal**: All three new API endpoints live and returning correct data.

#### Step 1.1 вЂ” DTOs

Create the six response DTOs in `GymScan.Services/Features/Progress/Dtos/Responses/`:

- `TrendDayDto(string Date, decimal Calories, decimal Protein, decimal Carbs, decimal Fat, bool HasData)`
- `TrendResponseDto(IReadOnlyList<TrendDayDto> Days, GoalSnapshotDto? Goals, int RangeInDays)`
- `DayStatusEntry(string Date, string Status, decimal TotalCalories)`
- `StreakResponseDto(int CurrentStreak, decimal GoalHitRate, IReadOnlyList<DayStatusEntry> HeatmapDays, bool HasGoal)`
- `WeekDayEntry(string Date, string DayName, decimal TotalCalories, int CaloriesTarget, bool HasData, string Status)`
- `WeeklySummaryResponseDto(string WeekStart, string WeekEnd, IReadOnlyList<WeekDayEntry> Days, decimal WeeklyTotal, decimal WeeklyGoal, bool HasGoal)`

Reuse the existing `GoalSnapshotDto` from `GymScan.Services/Features/Diary/Dtos/Responses/GoalSnapshotDto.cs`.

#### Step 1.2 вЂ” IProgressService and ProgressService

Create `GymScan.Services/Features/Progress/IProgressService.cs`:

```csharp
public interface IProgressService
{
    Task<ServiceResponse<TrendResponseDto>> GetTrendsAsync(int rangeInDays);
    Task<ServiceResponse<StreakResponseDto>> GetStreaksAsync();
    Task<ServiceResponse<WeeklySummaryResponseDto>> GetWeeklySummaryAsync(DateOnly weekStart);
}
```

Create `GymScan.Services/Features/Progress/ProgressService.cs` implementing the interface. Each method follows the DiaryService pattern: inject `AppDbContext _db` and `ICurrentUserService _currentUser`, resolve userId, query, and return `ServiceResponse<T>`.

**GetTrendsAsync(int rangeInDays)**:
1. Validate `rangeInDays` в€€ {7, 30, 90}; return 400 otherwise.
2. Compute `startDate` = today в€’ rangeInDays + 1 day; `endDate` = today.
3. Fetch all `MealLog` entries for userId in [startDate, endDate] via `AsNoTracking`.
4. Fetch active `DailyNutritionGoal` for userId.
5. Group entries by `DiaryDate`, sum macros per day.
6. Build one `TrendDayDto` per calendar day in the range (fill HasData=false for days with no entries).
7. Return `TrendResponseDto` ordered oldest в†’ newest.

**GetStreaksAsync()**:
1. Compute `startDate` = today в€’ 29 days; `endDate` = today (30 days inclusive).
2. Fetch all `MealLog` entries in [startDate, endDate] for userId.
3. Fetch active goal.
4. Build 30 `DayStatusEntry` items (oldest в†’ newest), classifying each day:
   - `NoData` вЂ” no entries.
   - `OnGoal` вЂ” total calories в‰Ґ goal Г— 0.75 AND в‰¤ goal.
   - `UnderGoal` вЂ” total calories < goal Г— 0.75.
   - `OverGoal` вЂ” total calories > goal.
5. Compute `CurrentStreak`: walk the list from newest toward oldest; count consecutive `OnGoal` days until a non-OnGoal day (NoData days do not break the streak; stop only on OverGoal or UnderGoal).
6. Compute `GoalHitRate`: `(OnGoal count / days with data count) Г— 100`. If no days with data, return 0.
7. Set `HasGoal = goal != null`.

**GetWeeklySummaryAsync(DateOnly weekStart)**:
1. If `weekStart.DayOfWeek != DayOfWeek.Monday`, return 400.
2. `weekEnd` = weekStart + 6 days.
3. Fetch all `MealLog` entries in [weekStart, weekEnd] for userId.
4. Fetch active goal.
5. Build 7 `WeekDayEntry` items (Mon в†’ Sun), classifying each day.
6. Compute `WeeklyTotal` = sum of calories for days with HasData.
7. Compute `WeeklyGoal` = `CaloriesTarget Г— (count of HasData days)`.
8. Return `WeeklySummaryResponseDto`.

### Phase 2 вЂ” Frontend: Progress Page and Components

**Goal**: Users can navigate to `/progress`, see today's progress, trend charts, streaks, and weekly summary.

**Prerequisites**: Phase 1 complete (API endpoints live).

#### Step 2.1 вЂ” API Client

Create `MacrosTrackerWeb/src/api/progressClient.js`:

```js
import client from "./client";

export const getProgressTrends = (range = 7) =>
  client.get(`/Progress/trends?range=${range}`).then(r => r.data.data);

export const getProgressStreaks = () =>
  client.get("/Progress/streaks").then(r => r.data.data);

export const getWeeklySummary = (weekStart) =>
  client.get(`/Progress/weekly${weekStart ? `?weekStart=${weekStart}` : ""}`).then(r => r.data.data);
```

The daily progress view reuses `getDiaryDay` from the existing `diaryClient.js`.

#### Step 2.2 вЂ” MacroProgressBar Component

Create `MacrosTrackerWeb/src/components/MacroProgressBar.jsx`.

Props: `label`, `consumed` (number), `goal` (number), `unit` (string, default "g").

Renders:
- Label (e.g., "Protein")
- Progress bar fill: `Math.min(consumed / goal, 1) * 100%`
- If `consumed <= goal`: green/neutral fill, shows `(goal в€’ consumed) unit remaining`
- If `consumed > goal`: red fill at 100%, shows `(consumed в€’ goal) unit over`
- Numeric display: `consumed / goal unit`

#### Step 2.3 вЂ” TrendChart Component

Create `MacrosTrackerWeb/src/components/TrendChart.jsx`.

Props: `days` (array of `TrendDayDto`), `metric` ("calories"|"protein"|"carbs"|"fat"), `goal` (number), `label` (string).

Uses Recharts `BarChart` or `LineChart`. Renders:
- X-axis: date labels (abbreviated, e.g., "Mon 26").
- Y-axis: auto-scaled.
- Data bars/line for the selected metric.
- `ReferenceLine` at `y={goal}` with dashed style labelled "Goal".
- No data message overlay when all `hasData = false`.

#### Step 2.4 вЂ” GoalHeatmap Component

Create `MacrosTrackerWeb/src/components/GoalHeatmap.jsx`.

Props: `days` (array of `DayStatusEntry`, 30 items).

Renders a CSS grid of 30 coloured tiles:
- `OnGoal` в†’ green
- `OverGoal` в†’ red
- `UnderGoal` в†’ amber
- `NoData` в†’ grey

Each tile shows the day number (1вЂ“31 of the month). A legend below the grid shows the four colours.

#### Step 2.5 вЂ” WeeklySummaryTable Component

Create `MacrosTrackerWeb/src/components/WeeklySummaryTable.jsx`.

Props: `summary` (`WeeklySummaryResponseDto`).

Renders a 7-row table: Day | Logged Calories | Goal | Status icon.
A footer row shows weekly total vs. weekly goal.
Previous/Next week navigation buttons are handled in the parent `Progress.jsx`.

#### Step 2.6 вЂ” Progress Page

Create `MacrosTrackerWeb/src/pages/Progress.jsx`.

Structure:
- Four tab buttons: "Today", "Trends", "Streaks", "Weekly"
- **Today tab** (active by default):
  - Calls `getDiaryDay()` from `diaryClient.js` (today).
  - Renders `<MacroProgressBar>` for calories, protein, carbs, fat.
  - Empty state when `dailySummary.totalCalories == 0`: "No food logged today. Head to Log Food to start tracking."
  - No-goal state when `goals == null`: "Set your nutrition goals to see progress." with a link to `/goal-setup`.
- **Trends tab**:
  - Range selector buttons: 7d / 30d / 90d (default 7d).
  - Calls `getProgressTrends(range)`.
  - Renders four `<TrendChart>` components (calories, protein, carbs, fat).
- **Streaks tab**:
  - Calls `getProgressStreaks()`.
  - Shows: current streak counter, goal hit rate percentage, `<GoalHeatmap>`.
  - No-goal state: "Set goals to track your streak."
- **Weekly tab**:
  - `weekOffset` state (0 = current week).
  - Computes `weekStart` from offset, calls `getWeeklySummary(weekStart)`.
  - Renders `<WeeklySummaryTable>` with prev/next navigation.
  - Disables "Next" when `weekOffset == 0`.
